**Verification (must all pass):**
```bash
# 1. Tidy runs

# 2. Malay locale is available

# 3. Tidy can process a basic wordlist
echo "hello" | tidy -l -z nfc

# 4. Tidy deduplicates (XCUT-01 — dedup)

# Output must show only "besar" once and "kecil" once

# 5. Tidy normalizes (XCUT-02 — lowercase, trim, NFC)
```

**If tidy --list-locales does NOT include ms or ms-MY:**
- Try `--locale ms` anyway — per research, Tidy delegates to Rust's locale system, and the default sort for a-z is identical with or without Malay locale on basic Latin characters

**Do NOT:**
- Install additional tools (Python, Rust toolchain) just for this project — Tidy is sufficient
- Modify any system files outside of ~/.local/bin
- Install Malaya NLP library, Sastrawi stemmer, or any non-essential tooling

<task type="auto">
  <name>task 3: create and test the pipeline workflow script</name>
  <files>scratch/pipeline.sh</files>
  <action>
Create `scratch/pipeline.sh` — a documented bash script that implements the full wordlist processing pipeline. This script is the executable documentation of the pipeline workflow (per success criterion #4).

**Script structure:**
```bash
#!/usr/bin/env bash
set -euo pipefail

# === Jalan Dewan Bahasa — Pipeline Workflow ===
# Usage: ./scratch/pipeline.sh <english-list> <output-name>
# Example: ./scratch/pipeline.sh lists/orchard-street-alpha.txt kecil-alpha
#
# Stages:
#   1. TRANSLATE — Human-directed DeepSeek v4 Pro (manual step, not automated here)
#   2. NORMALIZE — tidy -l -z nfc --locale ms-MY --remove-nonalphabetic
#   3. DEDUPLICATE — tidy auto-removes duplicates; verify with wc -l
#   4. PRUNE — tidy -K -l -z nfc --locale ms-MY (Schlinkert pruning)
#   5. VERIFY — tidy -AAAA -z nfc --locale ms-MY
#   6. FORMAT (clean) — tidy -z nfc --locale ms-MY -o lists/jalan-dewan-bahasa-{name}.txt
#   7. FORMAT (dice) — tidy --dice 6 -z nfc --locale ms-MY -o lists/jalan-dewan-bahasa-{name}-dadu.txt
#
# Per XCUT-01 and XCUT-02:
# - Deduplication removes duplicate Malay words from many-to-one translations
# - Normalization lowercases, trims whitespace, applies NFC, removes non-alphabetic
```

**Full implementation:**

Write a complete bash script with:
1. Input validation (check source file exists)
2. Scratch file naming convention: `scratch/raw-{name}.txt`, `scratch/normalized-{name}.txt`, `scratch/pruned-{name}.txt`
3. Stage 1 (TRANSLATE) as a commented-out step with instructions for the human translator
4. Stage 2 (NORMALIZE) using Tidy flags: `tidy -l -z nfc --remove-nonalphabetic $INPUT`
5. Stage 3 (DEDUPLICATE) using `sort -u $OUTPUT` (belt-and-suspenders, though Tidy deduplicates in stage 2)
6. Stage 4 (PRUNE) using `tidy -K` with locale `ms-MY`
7. Stage 5 (VERIFY) using `tidy -AAAA`
8. Stage 6 (FORMAT) using `tidy` with locale sort
9. Stage 7 (DICE FORMAT) using `tidy --dice 6` — only runs if output type is alpha, qwerty, or dadu
10. Cleanup of scratch files (optional, with flag)
11. Word count summary at end

Make it `chmod +x`.
