# Phase 3: Medium & Diceware Lists - Context

**Gathered:** 2026-07-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Translate two independent English wordlists into Standard Malay (DBP orthography) through the proven 7-stage pipeline: medium list (8,192 words) → `jalan-dewan-bahasa-sederhana.txt`, diceware list (7,776 words) → `jalan-dewan-bahasa-dadu.txt` (with 5-dice-roll prefixes) + `jalan-dewan-bahasa-dadu-bersih.txt` (clean variant). Also produce SQLite databases for each: `db/jalan-dewan-bahasa-sederhana.db`, `dadu.db`, `dadu-bersih.db`. All outputs must pass `tidy -AAAA` verification and meet word count targets.

No dice variant for medium list (confirmed in architecture docs). Each list is pipelined independently — no cross-list merging.

**Requirements:** SEDE-01, DADU-01, DADU-02, SQLI-05, SQLI-06, SQLI-07
</domain>

<decisions>
## Implementation Decisions

### Pruning Fallback at Scale

- **D-01:** If Tidy `-K` (Schlinkert pruning) fails to reach target word count for either list, try `-P` (prefix removal) and `-S` (suffix removal) as alternatives. Pick whichever strategy preserves the most words while maintaining unique decodability. Test on medium list first (8,192 words) before diceware.
- **D-02:** Do NOT proactively exclude short function words before pruning. Let Tidy handle prefix conflicts algorithmically.
- **D-03:** Only supplement from an external Malay frequency list as a last resort if `-K`, `-P`, and `-S` all fail to reach target.

### Translation Quality at Scale

- **D-04:** Translate each entire English list in a single batch via DeepSeek v4 Pro using the same system prompt established in Phase 2.
- **D-05:** After translation, spot-check 5–10% of each list (~400–800 words per list) for: Indonesian spellings, unnatural Malay, profanity, and DBP orthography violations. Flagged words are corrected individually and the list re-pipelined.
- **D-06:** The DBP orthography audit (grep for Indonesian patterns) from Phase 2 applies automatically to all words — this catches a large class of issues without manual review.

### List Independence

- **D-07:** Medium and diceware are translated and pipelined independently. Same English→Malay word may appear in both lists. This is acceptable — each list is consumed independently and UD is per-list.
- **D-08:** No cross-list deduplication or coordination.

### Diceware Clean Variant

- **D-09:** `jalan-dewan-bahasa-dadu-bersih.txt` is generated by stripping the tab-separated dice-roll prefix from `jalan-dewan-bahasa-dadu.txt` via `cut -f2`. This guarantees identical word sets.
- **D-10:** The clean variant passes through Tidy for sorting and locale-aware formatting before committing.

### OpenCode's Discretion

- Exact chunk size for DeepSeek batch translation (entire list or sub-batches)
- Pipeline execution order (medium first or diceware first)
- SQLite schema details beyond the established `words(roll, word)` table pattern
- Exact spot-check sampling methodology
</decisions>

## Existing Code Insights

### Reusable Assets
- `scratch/pipeline.sh` — Full 7-stage pipeline script. Extends with `--dice 6` for diceware list. Adding new output names for medium (`sederhana`) and diceware (`dadu`, `dadu-bersih`) is straightforward.
- `scratch/profanity-reject-ms.txt` — 33-entry curated Malay profanity reject list from Phase 2. Reusable as-is.
- `scratch/dbp-audit-kecil-alpha.txt` — DBP audit pattern and known Indonesian→Malaysian spelling mappings from Phase 2.

### Established Patterns
- Pipeline command pattern: `tidy <flags> -o <output> <input>` with per-stage flag variations
- Dice generation: `tidy --dice 6 <other flags> -o <output>.txt <pruned-input>`
- SQLite generation: `sqlite3 <db> "CREATE TABLE words(roll TEXT, word TEXT);"` + `.import`
- Profanity screening: `grep -Fxf profanity-reject-ms.txt <wordlist>`
- DBP audit: `grep -nE '(itas|sih|karena|universitas|nopember|agustus)\b' <wordlist>`

### Integration Points
- Medium list output: `lists/jalan-dewan-bahasa-sederhana.txt` + `db/jalan-dewan-bahasa-sederhana.db`
- Diceware list output: `lists/jalan-dewan-bahasa-dadu.txt` (with prefixes) + `lists/jalan-dewan-bahasa-dadu-bersih.txt` (clean) + `db/jalan-dewan-bahasa-dadu.db` + `db/jalan-dewan-bahasa-dadu-bersih.db`
- Pipeline scratch files follow the naming convention: `scratch/{raw,normalized,deduped,pruned}-{name}.txt`
