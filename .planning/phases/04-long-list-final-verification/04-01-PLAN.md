**Source English long list** (`lists/orchard-street-long.txt`):
- 17,576 lines, one lowercase English word per line, alphabetically sorted
- Highest-entropy list — target: 17,576 Malay words, ~14.1 bits/word entropy

- Example words: abandon, abandoned, abbey, abbot, abdomen, abduct, ability...

**Pipeline commands** (from Phase 3, proven at scale):
- Normalize: `tidy -l -z nfc --locale ms --remove-nonalphabetic -o scratch/normalized-besar.txt scratch/raw-besar.txt`
- Dedup: `sort -u scratch/normalized-besar.txt > scratch/deduped-besar.txt`
- Schlinkert prune: `tidy -K -l -z nfc --locale ms -o scratch/pruned-besar.txt scratch/deduped-besar.txt`
- Verify: `tidy -AAAA -z nfc --locale ms scratch/pruned-besar.txt`
- Whittle: `tidy --whittle-to 17576 -z nfc --locale ms -o lists/jalan-dewan-bahasa-besar.txt scratch/pruned-whittled-besar.txt`
- Format: `tidy -z nfc --locale ms -o lists/jalan-dewan-bahasa-besar.txt scratch/pruned-whittled-besar.txt`

**Pruning fallbacks** (per Phase 3 CONTEXT.md D-01 — applies to long list too):
- `tidy -P -l -z nfc --locale ms` (prefix removal) — try if -K loses too many words
- `tidy -S -l -z nfc --locale ms` (suffix removal) — try if -K loses too many words
- Pick whichever preserves the most words while maintaining unique decodability

**Phase 3 lessons learned** (critical for long list planning):
- **Medium (8,192):** 10.9% loss to normalization, 2.0% loss to Schlinkert pruning, 1,519 supplementary words needed
- **Diceware (7,776):** 32% dedup loss (many-to-one English→Malay mappings), 7.2% loss to pruning, 4,200+ supplementary words needed across 8 batches
- **Pattern:** Expect significant word loss from dedup (more vocabulary overlap at 17K scale) and moderate loss from pruning. Supplementation will be substantial — plan for multiple rounds.
- **Supplementary word categories that worked well:** Common root words, derived forms (ber-, meN-, peN-, ke-an, per-an), Arabic loanwords, scientific/technical vocabulary, archaic/classical terms, words starting with rare letters
- **SQLite pattern:** Python's built-in `sqlite3` module used when CLI `sqlite3` unavailable — same schema as existing DBs

**Profanity screening** (`scratch/profanity-reject-ms.txt`):
- 33-entry curated Malay profanity reject list from Phase 2, reused in Phase 3
- Method: `grep -Fxf scratch/profanity-reject-ms.txt lists/jalan-dewan-bahasa-besar.txt`
- DO NOT recreate this file — use the existing authoritative list

**DBP audit patterns** (from Phase 2-3):
